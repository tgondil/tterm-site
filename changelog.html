Update · v0.2.0

The week tterm learned to work for you
and to improve itself, safely.

Released 2026-07-24 · v0.2.0

A built-in assistant that actually does the work, a browser agent that's fast and signs into Google, click-to-scope self-improvement with a version-control safety net, and a reliability sweep underneath it all.

Assistant

A workspace assistant that does the work

  • Lives in the sidebar (⌘⇧J); sees your project, terminals, changes, and browser, and edits, runs, and drives to finish tasks.
  • One-click Sign in to Claude with auto-retry the moment login lands.
  • Durable project memory, an authoritative environment block (so it never asks your timezone), and markdown replies.
  • Leaner tokens: per-message context capped on a fresh run, a compact delta on follow-ups.
Browser agent

Fast, and it signs into Google

  • browser_act runs a whole sequence in one call — one model round-trip instead of one per action.
  • Adaptive settle: a same-page action returns in ~130ms instead of a fixed 320ms floor.
  • The embedded browser presents as real Google Chrome (UA + client hints), fixing the "this browser may not be secure" wall.
  • Chrome logins auto-import on launch; the launch toast reports your real Google sign-in state.
Self-improvement

Point at any part of tterm to improve it

  • Click-to-scope picker (⌘⇧E): the component under your cursor is resolved to its source files and handed to the assistant as a tight scope.
  • Containerized change: verify-self builds without touching the running app; one real build hot-reloads atomically.
  • Version-control safety rail: a scoped self-change snapshots the tree first, so it's undoable via Revert Last Self-Change (⌘K).
  • Reloads are gated on a completed build, so a bad change degrades to a recovery page instead of a white screen.
Craft

Live word-art, and a mascot that sails

  • A physics-driven word-art engine: sidebar backdrops, capsized-pane error art, themed typed seascapes.
  • A typed-sailboat mascot in the assistant header — it drifts when idle and sails hard while a task runs.
  • The motion above this page is that same engine, running live.
Foundations

Reliability & release identity

  • Atomic state writes so a crash can't corrupt your session or memory; corrupt memory is quarantined, not erased.
  • Unhandled errors are logged, not fatal; browser tools cap at 30s; download and tab races fixed.
  • The explorer remembers open folders per project across launches.
  • tterm now reports its version, commit, and build time — About tterm (⌘K) and a badge on the empty state open this page.
